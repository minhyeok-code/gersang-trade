# 거상 — 사용자 등급 및 EXP 정책

> **파일명:** `georang_grade_policy.md`
> **최종 수정일:** 2026-03-24
> **상태:** 초안 (Draft)

---

## 1. 개요

본 문서는 거상 서비스의 사용자 등급 체계, EXP 획득 규칙, 승급 구조, 매너점수 정책을 정의합니다. 등급 체계는 전통 상인(商人) 컨셉을 기반으로 하며, 사용자가 떠돌이 행상에서 시작해 최고의 거상으로 성장하는 서사를 담고 있습니다.

---

## 2. 등급 구조

등급은 총 **5단계**로 구성되며, 각 단계는 전통 한국 상인 직함에서 명칭을 가져옵니다. 최고 등급을 제외한 각 등급 내에서는 **호봉**을 통해 단계적으로 성장한 뒤 다음 등급으로 승급합니다.

| 등급 | 명칭 | 의미 | 단위 | 단계 수 | 단계당 필요 EXP | 등급 총 필요 EXP |
|------|------|------|------|--------|--------------|----------------|
| 5등급 | 행상 (行商) | 떠돌아다니는 장수 | 패 (牌) | 3패 | 50 | 150 |
| 4등급 | 보상 (褓商) | 보따리를 이고 다니는 상인 | 패 (牌) | 5패 | 150 | 750 |
| 3등급 | 객상 (客商) | 각지를 돌며 거래하는 상인 | 좌 (座) | 7좌 | 400 | 2,800 |
| 2등급 | 대상 (大商) | 큰 상인 | 방 (房) | 10방 | 1,530 | 15,300 |
| 1등급 | 거상 (巨商) | 상인 중의 제왕 | — | — | — | 누적 20,000 EXP 진입 |

> **설계 원칙:** 등급이 오를수록 단계 수와 단계당 필요 EXP가 동시에 증가하여 이중 지수형 난이도 곡선을 형성합니다. 대상 구간이 거상 진입 전 가장 긴 관문 역할을 합니다.

---

## 3. EXP 획득 규칙

### 3-1. 거래 확정 시 EXP 지급

EXP는 **양 당사자가 거래를 확정한 시점**, 즉 거래 성사 테이블이 생성될 때에만 지급됩니다. 미확정 또는 진행 중인 거래는 EXP를 부여하지 않습니다.

#### 거래 확정 기본 EXP

| 항목 | EXP |
|------|-----|
| 거래 확정 (기본) | +20 EXP |

#### 거래 금액별 추가 EXP

| 거래 금액 구간 | 추가 EXP |
|-------------|---------|
| 1억 미만 | +10 EXP |
| 1억 ~ 5억 | +20 EXP |
| 5억 ~ 15억 | +35 EXP |
| 15억 ~ 50억 | +55 EXP |
| 50억 이상 | +80 EXP |

> **설계 원칙:** 고가 거래일수록 더 많은 EXP를 획득하지만 상한선을 두어 격차를 제한합니다. 서비스 활성화의 핵심은 거래 금액이 아닌 거래 빈도이므로, 저가 다빈도 거래로도 충분히 성장할 수 있는 구조입니다.

#### 목표 난이도 기준

| 기준 항목 | 수치 |
|---------|------|
| 평균 거래 구간 | 1억 ~ 5억 |
| 평균 거래당 획득 EXP | 약 40 EXP |
| 거상 달성 목표 거래 횟수 | 약 500건 |
| 거상 진입 누적 필요 EXP | 20,000 EXP |

### 3-2. 클리어타임 기록 시 EXP 지급

사냥 허브 데이터 기여 보상. **거래 확정 EXP(§3-1)와 별도 경로**이며, 일일 상한도 **클리어타임에만** 적용한다.  
상세 UX·스냅샷 정책은 [`docs/clear-time-hunt-hub.ko.md`](./clear-time-hunt-hub.ko.md) 참고.

| 항목 | EXP |
|------|-----|
| 클리어타임 저장 (1회) | +5 EXP |

**지급 제한 (클리어타임 전용)**

| 조건 | EXP |
|------|-----|
| 동일 몬스터 + 동일 덱 스냅샷(`content_hash`) 재저장 | 0 (기록은 저장 가능) |
| 클리어타임 EXP 당일 지급 횟수 상한 초과 (3회/일, KST) | 0 |
| 클리어타임 6초 미만 또는 27초 이상 | 저장 거부 (EXP 없음) |

> 거래 확정·평가 EXP에는 **일일 상한 없음** (현재 정책).

## 4. 매너점수 시스템

### 4-1. 기본값 및 범위

| 항목 | 수치 |
|------|------|
| 신규 가입 기본 점수 | 60점 |
| 최고 점수 (상한) | 100점 |
| 최저 점수 (하한) | 0점 |

### 4-2. 평가 진행 흐름

1. 양측이 거래를 확정 → 거래 성사 테이블 생성 및 EXP 지급
2. 서버가 **SSE 알림**을 양 유저에게 발송
   > *"[거래명] 거래가 완료됐어요. 상대방을 3일 이내에 평가해보세요."*
3. 알림 클릭 시 **거래내역 페이지**로 이동하여 평가 진행
4. 평가는 자율 참여이며, 미평가에 대한 패널티 없음
5. 평가 가능 기간은 거래 확정 후 **3일**로 제한

### 4-3. 블라인드 평가 정책

보복 평가 방지를 위해 **블라인드 평가 방식**을 적용합니다.

```
거래 확정
    ↓
3일 평가 기간 시작
    ↓
각자 독립적으로 평가 제출
(제출 여부 및 평가 내용 상대방에게 비공개)
    ↓
3일 만료 시점에 양쪽 평가 동시 일괄 공개
(둘 다 제출 완료해도 만료 전까지 공개 안 함)
```

**세부 규칙**

| 항목 | 내용 |
|------|------|
| 평가 제출 중 | 상대방의 제출 여부 및 내용 열람 불가 |
| 3일 이내 한쪽만 제출 | 만료 시점까지 비공개 유지 |
| 3일 만료 후 | 제출된 평가만 공개 (미제출 시 평가 없음으로 처리) |
| EXP 및 매너점수 반영 | 3일 만료 시점에 일괄 반영 |

### 4-3. 평가 선택지 및 효과

| 평가 | EXP 효과 | 매너점수 효과 |
|------|---------|------------|
| 👍 좋음 | +15 EXP | +2점 |
| 😐 보통 | 0 EXP | 0점 |
| 👎 나쁨 | −20 EXP | −3점 |

> 부정 평가의 패널티를 긍정 평가의 보상보다 크게 설정하여 신뢰 거래를 유도합니다.

### 4-4. 프로필 공개 정보

매너점수는 거래량, 등급과 함께 상대방 프로필에 공개되어 거래 전 신뢰도 판단 기준으로 활용됩니다.

```
거래량    매너점수    등급
 128건     73점    객상 4좌
```

### 4-5. 동일 상대 반복 거래 모니터링

초기 단계에서는 제재 기능 없이 **관리자 탐지 및 모니터링**만 적용합니다.

| 항목 | 내용 |
|------|------|
| 탐지 조건 | 동일 두 유저 간 **7일 이내 3건 이상** 거래 발생 시 |
| 조치 | 관리자 대시보드에 플래그(⚠️) 표시 및 해당 거래 내역 열람 가능 |
| 자동 제재 | 없음 (추후 도입 검토) |
| 목적 | 어뷰징 패턴 데이터 수집 및 향후 제재 정책 수립 근거 마련 |

> 탐지 조건(7일 / 3건)은 실제 운영 데이터를 기반으로 추후 조정할 수 있습니다.

## 5. 거래내역 페이지

거래내역 페이지는 과거 거래를 조회하고 평가를 제출하는 중심 화면입니다. 모달 방식 대신 페이지로 제공하여 사용자가 언제든 평가 기간 내에 자유롭게 접근할 수 있도록 합니다.

| 상태 | 화면 표시 |
|------|---------|
| 평가 대기 중 (3일 이내) | **평가하기** 버튼 활성화 |
| 평가 완료 | 내가 남긴 평가 결과 표시 |
| 평가 기간 만료 | "평가 기간이 종료되었습니다" 안내 |

## 6. 핵심 정책 결정 요약

| 항목 | 결정 내용 |
|------|---------|
| EXP 곡선 | 지수형 (단계 수 + 단계당 EXP 동시 증가) |
| EXP 리셋 | 없음 — 누적 합산 방식 |
| 환불/취소 시 EXP | 해당 없음 (환불 기능 미구현) |
| 평가 트리거 | 거래 확정 후 SSE 알림 발송 |
| 평가 방식 | 거래내역 페이지 (모달 아님) |
| 평가 기한 | 거래 확정 후 3일 |
| 평가 참여 | 자율 |
| 매너점수 상한/하한 | 100점 / 0점 |
| 블라인드 평가 | 적용 — 3일 만료 시점 일괄 공개 |
| 어뷰징 방지 | 블라인드 평가 적용, 추가 정책 추후 정의 |
