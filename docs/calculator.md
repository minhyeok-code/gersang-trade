# 가성비 계산 기능 기획

## 개요

거상 아이템 거래 사이트의 부가 기능으로, 유저가 현재 보유한 스펙을 기준으로
어떤 아이템 또는 용병을 추가했을 때 데미지 상승 효율이 가장 높은지 계산해주는 기능이다.

---

## 핵심 개념

거상의 데미지는 두 가지 수치에 의해 결정된다.

- **저항 (Resistance)**: 몬스터가 가진 방어 수치. 저항이 높을수록 피해를 덜 받는다.
  디버퍼 용병이나 장비로 몬스터의 저항을 깎아 유효 데미지를 높인다.
- **속성값 (Attribute Value)**: 용병과 몬스터 각각이 보유한 수치.
  용병의 속성값이 몬스터 속성값에 비해 높을수록 추가 데미지가 발생한다.

---

## 데미지 계산 공식

### 1. 저항 유효 데미지 통과율

```
깎은 뒤 저항 = 몬스터 저항 - 총 디버프량

if 깎은 뒤 저항 >= 260:
    통과율 = 1.4%  (상한선 고정)

if 깎은 뒤 저항 < 260:
    통과율(%) = 100 - (깎은 뒤 저항 × 0.16 + 57)

통과율 최솟값 = 0 (음수 방지 처리 필요)
```

> 저항 260이 상한선으로, 260 이상이면 무조건 1.4%만 통과한다.
> 260 미만으로 깎을수록 통과율이 0.16%씩 선형으로 증가한다.

### 2. 속성 보정

```
속성 보정(%) = clamp((3 × 용병 속성값 - 몬스터 속성값) / 2, -50, +50)
```

- 최솟값: -50%
- 최댓값: +50%
- 상성 관계가 없는 속성 간 공격 시 데미지 증감 없음 (보정 0 처리)

### 3. 종합 데미지 배율

```
총 데미지 배율 = (100 + 속성 보정) × (통과율 / 100)
```

### 4. 현재 대비 데미지 상승률

스펙업 전후를 비교하는 공식이다.

```
데미지 상승률(%) = (총 데미지 배율_after / 총 데미지 배율_before - 1) × 100
```

### 5. 저깎 0% 기준 배율

현재 스펙이 저항깎 없는 상태 대비 몇 배인지 계산한다.

```
저깎 0% 기준 통과율 = 1.4%  (몬스터 저항 >= 260인 경우)
저깎 0% 기준 통과율 = 100 - (몬스터 저항 × 0.16 + 57)  (몬스터 저항 < 260인 경우)

저깎 0% 기준 배율 = 현재 총 데미지 배율 / 저깎 0% 기준 총 데미지 배율
```

---

## 가성비 점수 공식

```
가성비 점수 = 데미지 상승률(%) / 아이템 획득 비용(만 골드)
```

- 점수가 높을수록 같은 비용 대비 데미지 향상 효율이 좋음
- 리스트는 가성비 점수 내림차순 정렬
- 최고 점수 항목에 "추천" 표시

---

## 유저 입력 항목

유저가 직접 입력해야 하는 값들이다.

| 항목 | 설명 |
|------|------|
| 총 저항깎 수치 | 현재 보유한 디버퍼/장비의 저항 감소량 합산 |
| 내 용병 속성값 | 현재 용병의 속성 수치 |
| 몬스터 저항 | 목표 몬스터의 저항 수치 (유저 직접 입력) |
| 몬스터 속성값 | 목표 몬스터의 속성 수치 (유저 직접 입력) |

> 몬스터 데이터는 별도 DB 구축이 어려워 유저가 직접 입력한다.

---

## 가성비 체크 리스트

### 저항깎 아이템/용병 리스트

유저의 현재 저항깎에 해당 아이템/용병을 추가했을 때의 효과를 계산한다.

| 표시 항목 | 설명 |
|----------|------|
| 이름 | 아이템 또는 용병명 |
| 저항깎 수치 | 해당 아이템/용병이 제공하는 저항 감소량 |
| 가격 | 집계 테이블 기준 자동 입력, 유저 수정 가능 |
| 추가 후 저항 통과율 | 해당 아이템 추가 시 새 통과율 |
| 데미지 상승률 | 현재 대비 데미지 상승 % |
| 가성비 점수 | 데미지 상승률 / 가격(만 골드) |

### 속성값 아이템 리스트

유저의 현재 속성값에 해당 아이템을 추가했을 때의 효과를 계산한다.

| 표시 항목 | 설명 |
|----------|------|
| 이름 | 아이템명 |
| 속성 증가량 | 해당 아이템이 제공하는 속성값 증가 수치 |
| 가격 | 집계 테이블 기준 자동 입력, 유저 수정 가능 |
| 추가 후 속성 보정 | 해당 아이템 추가 시 새 속성 보정값 |
| 데미지 상승률 | 현재 대비 데미지 상승 % |
| 가성비 점수 | 데미지 상승률 / 가격(만 골드) |

---

## 가격 처리 방식

- 기본값: `material_price_history` 집계 테이블의 직전 달 평균가 자동 입력
- 유저가 직접 수정 가능 (한 달 내 시세 변동 반영 목적)
- 수정값은 해당 세션에서만 유효하며 DB에 저장하지 않음

---

## 공식 검증 결과

- 몬스터 저항 365, 저항깎 136 적용 시
  - 깎은 뒤 저항: 229
  - 저항 통과율: 100 - (229 × 0.16 + 57) = 6.36% → 게임 정보 사이트 표기 6.4%와 일치 (반올림 차이)
  - 저깎 0% 대비 배율: 6.36 / 1.4 = 4.54배 → 게임 정보 사이트 표기 4.5배와 일치

## UI 흐름

```
[유저 입력]
 ├─ 현재 저항깎 수치
 ├─ 현재 용병 속성값
 ├─ 목표 몬스터 저항
 └─ 목표 몬스터 속성값
         ↓
[현재 데미지 현황 표시]
 ├─ 깎은 뒤 저항
 ├─ 저항 통과율
 ├─ 속성 보정
 ├─ 저깎 0% 대비 배율
 └─ 종합 데미지 배율
         ↓
[탭 전환]
 ├─ 저항깎 아이템/용병 리스트
 └─ 속성값 아이템 리스트
         ↓
[각 리스트]
 ├─ 가성비 점수 내림차순 정렬
 ├─ 최고 점수 항목 "추천" 배지 표시
 └─ 가격 칸 유저 직접 수정 가능
```

---

## 엣지 케이스 처리

| 상황 | 처리 방식 |
|------|----------|
| 깎은 뒤 저항 >= 260 | 통과율 1.4% 고정 |
| 깎은 뒤 저항이 음수 | 통과율 상한 별도 정의 필요 (추후 확인) |
| 속성 보정 범위 초과 | -50 ~ +50 clamp 처리 |
| 상성 관계 없는 속성 | 속성 보정 0으로 처리 |
| 가격이 0 또는 미입력 | 가성비 점수 계산 제외 |
| 데미지 상승률이 0 이하 | 가성비 점수 0 또는 음수 표시 (비추천) |
