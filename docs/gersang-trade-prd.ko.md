### 용어(간단 정리)
- **세트**: 여러 아이템(피스)을 함께 착용/구성하여 세트 효과가 적용되는 묶음
- **피스**: 세트를 구성하는 개별 아이템 단위(예: 5피스 세트)
- **주술**: 각 피스에 적용 가능한 강화/옵션(주술 적용 피스 수에 따라 주술 세트 효과 변화)
- **사통팔달**: 서버 전체가 볼 수 있는 메시지 기능을 하는 유료 아이템(거래 공지/홍보에 활용)

### 1) 개요
- **서비스 목적**: 게임 **`거상`** 유저 간 **아이템 거래를 더 쉽고 투명하게** 만들기 위한 거래 지원 서비스
- **배경 인사이트**:
  - 거래사이트가 잘 되는 게임: **게임 내 거래 UX/기능이 불친절**한 경우가 많음
  - 거래사이트가 없는/약한 게임: **게임 내 거래 시스템이 잘 설계**된 경우가 많음
  - `거상`은 고가 아이템/세트/강화(주술 포함) 등 **정보 복잡도**가 높아 외부 서비스가 거래 효율을 크게 개선할 여지가 큼
- **핵심 가치**:
  - **등록/조회/신청/확정**을 통해 “팔렸는지/샀는지” 상태를 명확히 함
  - 세트/주술 구조를 **가시성 있게 모델링**하여 비교·검색·등록 편의 제공
  - **시세(평균/최대/최소/거래량)** 제공으로 합리적 거래 지원
  - **신고/차단**을 통한 사기 억제 및 커뮤니티 신뢰 형성

### 2) 목표 및 성공 지표 (예시)
- **제품 목표**
  - 거래 상태가 명확한 안전한 거래 프로세스 제공
  - 복잡한 세트/주술 아이템도 빠르게 등록·검색·비교 가능
  - 최근 시세를 한 눈에 제공하여 정보 비대칭 완화
- **성공 지표(KPI) 예시**
  - 등록 전환율(조회 → 등록), 신청 전환율(조회 → 신청), 거래 확정률(신청 → 확정)
  - 신고 대비 제재 처리 시간, 재발(재신고) 비율
  - 시세 페이지 재방문율, 검색 성공률(검색 후 상세 진입)

### 3) 범위(Scope)
- **MVP 포함**
  - 아이템 등록/조회/검색/필터
  - 거래 신청 및 거래 확정(상태 관리)
  - 신고 기능 및 관리자 차단
  - 최근 시세 통계(기간 선택 + 평균/거래량 + 선택형 min/max)
  - 세트/주술(및 주술 세트) 정보 입력/표시 구조
- **추가기능(확장)**
  - 아이템 능력치 조회(세트/주술/주술 세트 포함) + “가성비” 지표
  - **속성값 데미지 계산기**: 현재 속성값 기준으로 아이템 구매 시 데미지 증가분(m%) 계산
  - 사통팔달 메시지 검색/저장/연동(거래 상태로 “판매완료/구매완료” 문제 해결)
- **명시적 제외(Out of Scope)**
  - **현금 ↔ 아이템** 거래 중개/결제/에스크로 (정책상 불가)
  - 게임 계정/캐릭터 거래
  - 게임 내 직접 거래를 대체하는 기능(클라이언트 조작 등)

### 4) 사용자 및 권한 모델
- **비로그인 사용자(게스트)**
  - 아이템 **등록 현황 조회 가능**
  - 검색/필터/시세 조회 가능
  - 거래 신청/확정/등록/신고는 불가(정책에 따라 신고는 선택: MVP에서는 로그인 필수 권장)
- **인가된 사용자(로그인 사용자)**
  - 아이템 등록
  - 거래 신청
  - 거래 확정
  - 신고 등록
- **관리자**
  - 신고 조회/처리(승인/반려/추가 조치)
  - 사용자 차단(기간/영구), 차단 해제
  - 콘텐츠 관리(부적절 게시글/등록물 숨김 등)

### 5) 서비스 정책(필수)
- **현금과 아이템의 거래는 불가**
  - 서비스 약관/가이드에 명시
  - 게시물/채팅/메모 영역에서 현금 유도 문구 탐지(확장) 또는 신고 대상으로 분류
- **공개 범위**
  - 아이템 등록 현황/목록: 모든 이용자 조회 가능
  - 등록/신청/확정: 로그인 사용자만 가능
- **윤리/안전**
  - 사기 등 비도덕적 행동은 **신고**로 관리자에게 전달
  - 관리자는 검토 후 **사용자 차단** 가능

### 6) 핵심 기능 상세

#### 6.1 아이템 등록(판매 글)
- **등록 단위**: “판매 아이템(단품 또는 세트)” + 옵션/강화/주술 상태 + 희망 조건(게임 내 재화 기준)
- **필수 정보 예시**
  - 서버(필수), 아이템명(필수)
  - 판매 유형: 단품/세트
  - 가격(게임 내 재화 단위), 수량(가능한 경우)
  - 연락 방식(게임 내 닉네임/연락 메모 등)
- **상태**
  - 등록됨(판매중) → 거래중(신청 수락 등) → 판매완료(거래 확정) / 취소

#### 6.2 거래 신청 및 확정
- **거래 신청**
  - 구매자가 특정 등록물에 대해 “거래 신청”
  - 판매자는 신청을 확인하고 진행 여부를 결정(수락/거절/대기)
- **거래 확정**
  - 거래가 완료되면 확정 처리하여 **판매완료/구매완료** 상태가 기록됨
  - 확정 기록은 **시세 통계** 산출의 근거 데이터가 됨

#### 6.3 신고/제재(관리)
- **신고 대상**
  - 사기 의심, 욕설/비방, 현금 유도, 허위 매물, 기타 비윤리 행위
- **신고 데이터**
  - 대상(사용자/등록물/거래), 사유 카테고리, 상세 내용, 증빙(스크린샷 등)
- **관리자 조치**
  - 경고/게시물 숨김/사용자 차단(기간 또는 영구)

### 7) 도메인: 세트/주술/주술 세트 모델링 요구사항

#### 7.1 배경
- `거상` 아이템은 강화옵션이 다양하고 **세트 효과**가 존재
- 세트의 각 피스는 **‘주술’ 강화**가 가능
- ‘주술’이 적용된 피스 수에 따라 **‘주술 세트’ 추가 능력치**가 적용됨
- 실제 거래 예:
  - 5피스 세트 기준: 2피스는 주술 없음, 3피스는 주술 적용(= 3주술 세트) 형태로 판매

#### 7.2 서비스 요구
- **등록 편의성**
  - “5피스 세트”를 한 번에 등록하되, 각 피스별로 **주술 여부/옵션**을 입력 가능
  - “주술 적용 피스 수(0~5)”가 한 눈에 보이도록 요약 표시
- **조회/가시성**
  - 목록 카드에서 최소 요약: 세트명, 피스 수, 주술 적용 수, 핵심 옵션
  - 상세 페이지에서 피스별 주술/옵션을 테이블로 보여주기
- **검색/필터**
  - 세트명, 피스 수, 주술 적용 수(예: 3/5), 특정 피스 주술 여부 등으로 필터 가능(확장)

### 8) 사통팔달(메시지) 검색 기능 방향(확장)
- **문제**
  - 고가 거래가 사통팔달로 이루어지나 시간이 지나면 메시지 추적이 어렵고
  - 해당 아이템이 실제로 팔렸는지 여부는 당사자에게 묻지 않으면 불명확
- **제공 가치**
  - 사통팔달 메시지를 검색/저장/인덱싱하여 “찾기” 문제를 해결
  - 거래 등록/확정 상태와 연결하여 **판매완료 여부가 시스템에 남도록** 함
  - 사통팔달은 유료아이템이자 게임 내 재화 성격이므로, 외부에서 추적/정리되면 유저의 비용 부담도 완화 기대

### 9) 시세(최근 평균 거래가) 기능
- **데이터 원천**
  - “거래 확정”된 거래 기록을 기반으로 통계 산출
- **기본 제공**
  - 기간 선택: **7일 / 15일 / 30일**
  - 표시 항목(기본): 거래 수량(거래 건수), 평균가
  - 선택 항목(토글): min / max
- **표현 방식(예시)**
  - 기간별 요약 카드 + 차트(선/막대) + 상세 테이블

### 10) 아이템 능력치 조회 및 가성비 지표(추가기능)

> 세부 기획은 `docs/value-for-money-feature.ko.md` 참고.
- **배경**
  - 거래 서비스 특성상 “아이템 성능 데이터”가 있으면 비교/의사결정이 쉬움
  - 최근 거래가와 유저가 중요시하는 능력치를 결합하면 “가성비”를 수치화 가능
- **기능**
  - 아이템 상세에 세트 효과/주술 효과/주술 세트 효과를 포함한 능력치 표시
  - 가성비 점수 표시(높을수록 좋음): (가중치 적용 능력치 합) ÷ 최근 평균가 형태의 지표 예시  
    - 방향성은 `docs/calculator.md`의 “효과/가격”과 동일
    - 예: “공격력 가중치 0.6 + 생존 가중치 0.4” 등 사용자 프리셋(확장)
  - 결과 표현: “능력치 1점당 00원” 또는 “주요 능력치당 00원” 같이 직관적 문구로 제공
- **주의**
  - 가성비는 해석이 갈릴 수 있어 “계산 방식”을 투명하게 공개하고, 가중치 커스터마이즈(확장)를 고려

### 11) 주요 화면(IA) 초안
- **홈**
  - 인기 검색어/최근 등록/시세 요약
- **아이템 목록**
  - 검색, 필터(서버/세트/주술 수/가격/상태), 정렬(최신/가격/거래임박 등)
- **아이템 상세**
  - 세트/피스/주술 정보, 판매 조건, 거래 신청, 신고
  - 시세 요약(해당 아이템)
- **등록**
  - 단품/세트 선택, 피스 구성, 주술 체크, 가격/메모 입력
- **마이페이지**
  - 내 등록물, 내 신청/진행 거래, 내 신고 내역
- **관리자**
  - 신고 큐, 사용자/게시물 관리, 차단 관리

### 12) 핵심 유저 시나리오(요약)
- **판매자**
  - 세트/주술 정보를 포함해 등록 → 구매 신청 수락 → 게임 내에서 거래 진행 → 사이트에서 거래 확정
- **구매자**
  - 조건에 맞는 아이템 검색 → 상세에서 세트/주술 확인 → 거래 신청 → 거래 완료 후 확정
- **관리자**
  - 신고 접수 → 증빙 확인 → 사용자 차단/게시물 숨김 → 처리 결과 기록

### 13) 비기능 요구사항(초안)
- **신뢰성**
  - 거래 확정 기록 무결성(감사 로그), 상태 전이 제한(예: 완료 후 수정 불가)
- **확장성**
  - 아이템/세트/주술 데이터 증가를 고려한 검색 인덱스 설계
- **보안**
  - 로그인 사용자만 쓰기 작업 가능(등록/신청/확정/신고)
  - 관리자 권한 분리 및 감사 로그

### 10-2) 속성값 데미지 계산기(추가기능)
- **배경**
  - 거상의 속성 데미지는 내 속성값(x)과 몬스터 속성값(y)의 차이에 의해 결정됨
  - 구매 후보 아이템의 속성값이 얼마나 실질적인 데미지 향상을 가져오는지 수치로 확인하고 싶은 니즈 존재
- **공식**
  - 추가 데미지 비율: `n% = (3x - y) / 2`
  - n의 범위: **-50% ~ +50%** (이 범위를 벗어나면 cap 적용)
  - 변수:
    - `x` = 내 속성값 (사용자 직접 입력)
    - `y` = 몬스터 속성값 (사용자 입력 또는 서비스 기준값 — 결정 필요)
    - `delta` = 구매 후보 아이템의 속성값 (`ItemStat.ELEMENT_VALUE`에서 조회)
- **계산 흐름**
  1. 사용자가 현재 속성값 `x`를 입력
  2. 구매 후보 아이템 선택 → 아이템의 `ELEMENT_VALUE = delta` 조회
  3. 아이템 구매 전 데미지 보너스: `n_before = clamp((3x - y) / 2, -50, 50)`
  4. 아이템 구매 후 데미지 보너스: `n_after = clamp((3(x + delta) - y) / 2, -50, 50)`
  5. **데미지 증가분**: `m = n_after - n_before` (단위: %)
- **표시 방식**
  - `이 아이템 구매 시 속성 데미지 +m%p 향상` 형태로 직관적 표기
  - n이 이미 +50 cap에 도달한 경우 "이미 최대치 도달" 안내 표시
- **데이터 요구사항**
  - 아이템의 속성값: `ItemStat.statType = ELEMENT_VALUE`, `element = 해당 속성` 행에서 조회
  - 몬스터 속성값(y): 오픈 이슈 (아래 14항 참고)

### 14) 오픈 이슈(결정 필요)
- **신고 권한**: 게스트 신고 허용 여부(스팸/악용 vs 접근성)
- **속성값 계산기 — 몬스터 속성값(y) 입력 방식**: 사용자가 직접 입력 vs 서비스에서 대표 몬스터 기준값 제공 vs y=0 고정(내 속성값만 기준으로 단순화)
- **연락 방식**: 게임 내 닉네임 공개 수준/개인정보 처리(가이드 필요)
- **거래 확정 방식**: 판매자/구매자 단독 확정 vs 상호 확정(분쟁 최소화 관점)
- **사통팔달 데이터 수집 방식**: 사용자 입력(복사/붙여넣기) vs 크롤링/연동(정책/기술/법적 검토)
