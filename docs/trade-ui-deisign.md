## 1. 레이아웃 원칙

- **데스크탑 기준 설계**: 거상은 PC 게임이고 거래는 게임 실행 중 병행하는 행위이므로 데스크탑 UX를 우선한다.
- **타겟 유저**: 거상 고인물 유저. 게임 용어(북두, 풀지국, 변 등)를 그대로 사용하며 과잉 친절한 설명 텍스트는 넣지 않는다.
- **재화 단위**: `전` 단일 사용 (예: 680억 전)

## 2. 전체 레이아웃 구조

```
┌──────────────────────────────────────────────────────┐
│  로고   거래 | 시세 | 전투 계산기        [알림] [채팅] │  ← 상단 네비
├──────────────────────────────────────────────────────┤
│         │                    │                       │
│  사이드바 │   팝니다 목록       │   삽니다 목록          │
│  (필터)  │                    │                       │
│         │                    │                       │
└──────────────────────────────────────────────────────┘
```

- 좌측: 필터 사이드바 (220px 고정)
- 중앙/우측: 팝니다 | 삽니다 좌우 분할 (각 50%)
- 상단 우측: 알림 아이콘, 채팅 아이콘

## 3. 사이드바 필터

### 3.1 필터 항목

| 필터 | 선택지 | 비고 |
|---|---|---|
| 구분 | 전체 / 단품 / 세트 | 탭 형태 |
| 세트명 검색 | 텍스트 입력 | 세트 선택 시만 노출 |
| 구성 | 전체 / 갑투 / 변 / 풀 / 풀반쌍 | 세트 선택 시만 노출 |
| 주술 | 전체 / 없음 / 천추 / 북두 / 천기 / 개양 ... | DB에서 해당 세트 적용 가능 주술만 노출 |
| 주술 수 | 전체 / 3셋 / 5셋 | 세트 선택 시만 노출 |

### 3.2 필터 유지 정책

- 팝니다 / 삽니다 탭 전환 시 필터 상태 유지
- URL 쿼리 파라미터로 필터 상태 관리
  ```
  /trade?type=set&name=지국천왕&composition=full&ritual=북두&pieces=5&intent=sell
  ```

### 3.3 주술 수 선택지 동적 변경

구성 선택에 따라 주술 수 최대값이 달라진다.

| 구성 | 주술 수 선택지 |
|---|---|
| 갑투 (2피스) | 전체 / 1셋 / 2셋 |
| 변 (3피스) | 전체 / 1셋 / 2셋 / 3셋 |
| 풀 / 풀반쌍 (6피스) | 전체 / 1셋 / 2셋 / 3셋 / 4셋 / 5셋 |
| 전체 | 전체 / 3셋 / 5셋 |

## 4. 거래 목록 (좌우 분할)

### 4.1 탭

- `[전체 | 팝니다 | 삽니다]` 탭으로 전환
- 전체 탭: 팝니다 / 삽니다 좌우 분할 노출
- 팝니다 / 삽니다 탭: 해당 목록만 전체 너비로 노출

### 4.2 정렬

팝니다 / 삽니다 각각 독립적으로 정렬 기준 선택 가능

- 최신순 (기본)
- 가격 낮은순
- 가격 높은순

## 5. 거래 카드 스펙

### 5.1 카드 구조

```
┌─────────────────────────────────────┐
│ [팝니다]               닉네임        │  ← 전체 탭일 때만 뱃지 노출
├─────────────────────────────────────┤  ← 구분선
│                                     │
│ 3북두 풀지국반쌍                     │  ← 주술수+주술명+구성명+세트명
│                                     │
│ 500억 전                            │
│ 게임닉: 홍길동 | 18시~24시           │
│                          1시간 전   │
└─────────────────────────────────────┘
```

### 5.2 카드 필드 정의

| 필드 | 설명 | 비고 |
|---|---|---|
| 뱃지 | 팝니다 / 삽니다 색상 구분 | 전체 탭에서만 노출. 팝니다=연초록, 삽니다=연파랑 |
| 닉네임 | 서비스 계정 닉네임 | 우상단 |
| 제목 | 주술수+주술명+구성명+세트명 자동 생성 | 예: `3북두 풀지국반쌍` |
| 가격 | 숫자 + `전` | 예: `500억 전` |
| 게임닉 | 게임 내 닉네임 | 등록 폼에서 별도 입력 |
| 접속시간 | 시작~종료 시간 | 드롭다운으로 구조화 입력 |
| 등록시간 | 상대 시간 | 예: `1시간 전`, `어제` |

### 5.3 제목 자동 생성 규칙

```
{주술수}{주술명} {구성명}{세트명}

예:
  주술 없음 → 풀지국
  북두 3셋  → 3북두 풀지국
  북두 5셋  → 5북두 풀지국반쌍
  갑투 북두 → 북두 갑투지국
```

## 6. 상세 모달

### 6.1 구조

```
┌─────────────────────────────────────────┐
│ 거래 상세                           [×] │
├──────────────┬──────────────────────────┤
│              │                          │
│  프로필 패널  │      거래물 정보          │
│  (180px)     │                          │
│              │                          │
├──────────────┴──────────────────────────┤
│                    [신고]   [채팅하기]   │
└─────────────────────────────────────────┘
```

### 6.2 프로필 패널 (좌측)

| 항목 | 내용 |
|---|---|
| 프로필 사진 | 업로드 이미지 또는 이니셜 대체 |
| 닉네임 | 서비스 계정 닉네임 |
| 등급 | 현재 유저 등급 뱃지 |
| 서버 | 게임 서버명 |
| 게임 닉네임 | 게임 내 닉네임 |
| 접속 시간 | 등록 시 입력한 접속 가능 시간 |

### 6.3 거래물 정보 (우측)

| 항목 | 내용 |
|---|---|
| 아이템 이미지 | 크롤링 이미지 또는 디폴트 이미지 |
| 팝니다/삽니다 뱃지 | 색상 구분 |
| 제목 | 카드와 동일 (자동 생성) |
| 가격 | 카드와 동일 |
| 구성 | 풀세트 + 반지쌍 등 풀어서 표기 |
| 주술 | 북두칠성 5셋 등 풀어서 표기 |
| 한줄 설명 | 최대 20자 자유 입력 |
| 등록 시간 | 상대 시간 |

### 6.4 하단 버튼

| 버튼 | 동작 |
|---|---|
| 신고 | 신고 모달 오픈 |
| 채팅하기 | 모달 닫힘 → 채팅 모달 오픈 |

### 6.5 내 매물 클릭 시

채팅하기 버튼 대신 `수정` / `삭제` 버튼 노출

## 7. 채팅

### 7.1 진입 경로

| 경로 | 동작 |
|---|---|
| 상세 모달 → 채팅하기 | 모달 닫힘 → 채팅 모달 바로 오픈 |
| 상단 채팅 아이콘 | 우측 사이드 패널에 채팅 목록 노출 → 채팅방 클릭 → 채팅 모달 오픈 |

### 7.2 채팅 모달

- 상대방과 1:1 채팅
- 채팅 모달 내에 거래 대상 아이템 정보 상단 고정 표시 (어떤 매물 관련 채팅인지 맥락 유지)

## 8. 시세 모달

### 8.1 진입

- 별도 시세 버튼 또는 상단 네비 `시세` 메뉴

### 8.2 구조

```
시세 조회

세트명   [지국천왕 검색]
구성     [갑투] [변] [풀] [풀반쌍]
주술     [없음] [천추] [북두] [천기] ...
주술 수  [3셋] [5셋]

━━━━━━━━━━━━━━━━━━━━━━━━━
풀 + 북두 + 5셋 기준  (최근 7일)

평균    520억 전
최고    680억 전
최저    410억 전
거래량  8건
━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 8.3 정책

- 필터와 시세 조건은 **독립적** (거래 목록 필터와 무관)
- 기간 선택: 7일 / 15일 / 30일
- 조건이 구체적일수록 의미 있는 시세 → 모달 안에서 자유롭게 조합 선택

## 9. 이미지 처리

### 9.1 아이템 이미지

| 상태 | 처리 |
|---|---|
| 크롤링 이미지 있음 | 해당 이미지 노출 |
| 크롤링 이미지 없음 | 디폴트 이미지 적용 |

### 9.2 디폴트 이미지 규칙

- 관리자가 디폴트 이미지 N개 업로드
- `아이템 ID % N` 으로 디폴트 이미지 고정 배정
- 동일 아이템은 누가 봐도, 새로고침해도 항상 같은 디폴트 이미지

### 9.3 프로필 이미지

| 상태 | 처리 |
|---|---|
| 업로드 이미지 있음 | 해당 이미지 노출 |
| 없음 | 닉네임 첫 글자 이니셜로 대체 |

## 10. 모바일 대응 범위

| 기능 | 모바일 대응 |
|---|---|
| 거래 목록 / 등록 / 채팅 | 데스크탑 기준. 모바일 최적화 후순위 |
| 시세 조회 | 모바일 대응 |
| 전투 계산기 | 모바일 대응 |
| 데미지 분석 | 모바일 대응 |

> 거상은 PC 게임이므로 거래 행위는 데스크탑 환경에서 이루어진다.  
> 모바일은 조회/계산 기능 위주로 최적화한다.
