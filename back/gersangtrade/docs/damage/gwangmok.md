# 각성 광목천왕 데미지 공식 역산 정리

## 실험 조건

* 몬스터:

    * 매우 강한 몬스터 기준
    * 속성값 최저치 (-50%)
    * 저항깎 미적용
* 데미지:

    * 서버 내부에서는 소수점 유지
    * 유저 표시 시 `floor()` (버림)
* 고위 용병 특성:

    * 최소 공격력 = 최대 공격력

## 분석

힘은:

1. 공격력으로 환산
2. 스킬 자체에도 직접 계수 존재

공격력 환산분:

```text
STR 100 → ATK +25
ATK +25 → Damage +1
```

하지만 실제 데미지는 평균적으로 약 +1.6~1.7 증가.

따라서 힘 직접 계수 존재 추정:

```text
STR coefficient ≈ 0.0065
```

즉:

```text
Damage += 0.0065 × STR
```


# 중요 사항

## 서버 내부 소수점 유지

거상은:

* 내부 계산 시 소수점 유지
* 최종 유저 표시 시 floor()

예시:

```text
95.999 → 95
96.000 → 96
```

따라서:

* 레벨업 직후 변화가 안 보일 수 있음
* 내부 누적 후 특정 시점에 데미지 +1 발생

# CLI 구현 권장 사항

## 추천 구현 구조

```ts
type DamageInput = {
    level: number;
    atk: number;
    str: number;
    hp: number;
}
```

---

## 계산 예시

```ts
function calculateDamage(input: DamageInput): number {
    const {
        level,
        atk,
        str,
        hp
    } = input;

    const damage =
        75
        + (0.04 * atk)
        + (0.0065 * str)
        + (0.076 * hp)
        + (0.03 * level);

    return Math.floor(damage);
}
```
